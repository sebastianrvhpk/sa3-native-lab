# Interface Review Rubric

## Triage Table

| Idea | User action | System capability | Artifact touched | Feedback | Test | Risk | Decision |
| --- | --- | --- | --- | --- | --- | --- | --- |

## Priority Levels

- P0: required for trust and operation.
- P1: makes the workflow fast and playable.
- P2: expands research cognition.
- P3: visual identity and feeling.
- P4: ornamental or unsupported.

## Acceptance Tests

For each kept interaction, define:

- What the user sees before action.
- What the user does.
- What state changes.
- What artifact is produced or updated.
- What metadata proves provenance.
- How failure is shown.

## Deletion Tests

Delete or defer an idea when:

- It has no code path.
- It cannot be tested.
- It hides uncertainty.
- It creates a false sense of control.
- It requires product-scale infrastructure before research value is proven.
