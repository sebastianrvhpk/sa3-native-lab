# Role Signal Map

Use this to shape a research prototype into a portfolio demo for creative AI, design engineering, or full-stack roles.

## Competency To Evidence

| Competency | Evidence artifact |
| --- | --- |
| React + TypeScript | Typed components, domain models, clean route structure |
| Component architecture | Reusable controls, clear composition, no fake abstractions |
| AI tool intuition | Interface maps to model objects and uncertainty |
| Research tooling | Reproducible runs, artifact provenance, experiment notes |
| Full-stack skill | Typed API boundary, schemas, persistence plan or implementation |
| Realtime/canvas curiosity | Real graph, lineage, collaboration, or canvas interaction |
| Design engineering | Strong visual system tied to workflow, accessible controls |
| Ambiguity handling | Explicit unknowns, staged roadmap, risk register |
| Performance/debugging | Profiling notes, batching strategy, heavy-model boundary |
| Open-source readiness | README, setup path, examples, tests, license clarity |

## Demo Narrative

1. Research question.
2. System architecture.
3. Key interaction.
4. Technical implementation.
5. What the experiment revealed.
6. What remains unknown.
7. What would be built next.

## Honesty Rules

- Say prototype when prototype.
- Say research probe when unreliable.
- Separate model capability from interface capability.
- Keep failed experiments in the story when they shaped the result.
